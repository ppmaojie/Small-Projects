**版本**: v1.0  
**创建日期**: 2026-03-20  
**状态**: 待开发

---

## 1. 产品概述

### 1.1 产品名称
**还活着么** (Alive Check)

### 1.2 产品定位
一款反向签到社交应用，用"死亡焦虑"治疗拖延症，用黑色幽默制造社交传播。

### 1.3 目标用户
- 18-35 岁年轻人
- 喜欢黑色幽默、段子文化
- 有拖延症、需要外部督促
- 活跃于社交媒体，愿意分享有趣内容

### 1.4 核心价值
- **对用户**: 用游戏化方式培养签到习惯，同时获得社交乐趣
- **对传播**: 讣告卡片自带话题性，容易引发病毒式传播

## 2. 功能需求

### 2.1 MVP 功能列表 (v1.0)

| 优先级 | 功能模块 | 功能描述 |
|--------|----------|----------|
| P0 | 用户注册 | 微信一键登录/手机号注册 |
| P0 | 存活倒计时 | 设置倒计时，首页展示剩余时间 |
| P0 | 签到续命 | 点击按钮重置倒计时 |
| P0 | 紧急联系人 | 绑定 1-3 个好友作为通知对象 |
| P0 | 讣告通知 | 倒计时归零后自动发送通知给联系人 |
| P1 | 墓地页面 | 查看"已故"好友列表 |
| P1 | 献花留言 | 给"已故"好友献花、留言 |
| P1 | 复活功能 | "死亡"后可申请复活 |
| P2 | 排行榜 | 存活天数、死亡次数排行 |
| P2 | 讣告分享 | 生成讣告卡片分享到朋友圈 |

### 2.2 功能详细说明

#### 2.2.1 用户注册/登录
- 支持微信小程序一键登录
- 支持手机号 + 验证码登录
- 注册时需设置昵称（支持 emoji）
- 可选上传头像

#### 2.2.2 存活倒计时
- 默认存活周期：24 小时
- 可选周期：12h / 24h / 48h / 72h / 自定义
- 首页醒目展示倒计时（时：分：秒）
- 倒计时最后 1 小时显示红色预警
- 倒计时最后 10 分钟开始震动提醒（小程序）

#### 2.2.3 签到续命
- 点击"我还活着"按钮重置倒计时
- 签到时可optional 填写"存活感言"（限 50 字）
- 连续签到有额外奖励（连续 7 天、30 天徽章）

#### 2.2.4 紧急联系人
- 可绑定 1-3 个紧急联系人
- 联系人必须是已注册用户
- 绑定需对方确认（防止骚扰）
- 可设置不同联系人的通知优先级

#### 2.2.5 讣告通知
- 倒计时归零后触发
- 通知渠道：小程序模板消息 / 短信
- 讣告文案支持预设模板 + 自定义
- 讣告包含"献花"链接，点击跳转到墓地页面

预设讣告模板：
```
1. 【标准版】您关注的好友 [XXX] 已超过 [XX] 小时未签到，系统判定其可能已...节哀。
2. 【打工人版】[XXX] 走了，带着没写完的 PPT 走了。
3. 【社畜版】[XXX] 终于不用再早起了，愿天堂没有打卡。
4. 【剁手版】[XXX] 走了，花呗还没还完。
5. 【自定义】用户自定义文案（限 100 字）
```

#### 2.2.6 墓地页面
- 展示所有"已故"好友列表
- 每个"逝者"显示：头像、昵称、死亡时间、死亡时长
- 可点击进入详情页献花、留言
- 留言支持公开/私密两种模式

#### 2.2.7 献花留言
- 献花：免费，每次献花增加 1 朵花
- 留言：限 100 字，支持 emoji
- 留言可点赞，热门留言置顶

#### 2.2.8 复活功能
- "死亡"后可随时申请复活
- 复活方式（任选其一）：
  - 简单模式：点击确认按钮
  - 正常模式：拍一张照片证明活着
  - 困难模式：录制 5 秒语音说"我还活着"
- 复活后通知所有献花的好友："XXX 诈尸了！"

#### 2.2.9 排行榜
- 最苟王者：连续存活天数 TOP100
- 反复横跳：死亡 - 复活次数 TOP100
- 最佳悼词：讣告被点赞数 TOP100
- 每周重置，上榜用户获得虚拟徽章

#### 2.2.10 讣告分享
- "死亡"后自动生成讣告卡片
- 卡片包含：逝者头像、昵称、死亡时间、讣告文案
- 一键分享到朋友圈/微信群
- 点击卡片可下载小程序

## 4. 数据模型

### 4.1 用户表 (users)

```javascript
{
  _id: ObjectId,
  openid: String,          // 微信 openid
  phone: String,           // 手机号
  nickname: String,        // 昵称
  avatar: String,          // 头像 URL
  createdAt: Date,
  updatedAt: Date
}
```

### 4.2 存活记录表 (alive_records)

```javascript
{
  _id: ObjectId,
  userId: ObjectId,        // 用户 ID
  cycleHours: Number,      // 存活周期（小时）
  lastCheckin: Date,       // 最后签到时间
  nextDeadline: Date,      // 下次截止时间
  status: String,          // alive/dead
  consecutiveDays: Number, // 连续签到天数
  totalDeaths: Number,     // 总死亡次数
  createdAt: Date,
  updatedAt: Date
}
```

### 4.3 联系人表 (contacts)

```javascript
{
  _id: ObjectId,
  userId: ObjectId,        // 用户 ID
  contactUserId: ObjectId, // 联系人用户 ID
  priority: Number,        // 通知优先级 1-3
  status: String,          // pending/confirmed
  createdAt: Date
}
```

### 4.4 讣告表 (obituaries)

```javascript
{
  _id: ObjectId,
  deceasedUserId: ObjectId, // 逝者用户 ID
  templateId: Number,       // 讣告模板 ID
  customText: String,       // 自定义文案
  deathTime: Date,          // 死亡时间
  notifiedContacts: [ObjectId], // 已通知的联系人
  createdAt: Date
}
```

### 4.5 墓地互动表 (graveyard_interactions)

```javascript
{
  _id: ObjectId,
  deceasedUserId: ObjectId, // 逝者用户 ID
  visitorUserId: ObjectId,  // 访客用户 ID
  type: String,             // flower/message
  content: String,          // 留言内容
  likes: Number,            // 点赞数
  isPrivate: Boolean,       // 是否私密
  createdAt: Date
}
```

### 4.6 排行榜表 (leaderboards)

```javascript
{
  _id: ObjectId,
  type: String,             // consecutive_days/total_deaths/best_obituary
  userId: ObjectId,
  score: Number,
  week: Number,             // 第几周
  rank: Number,
  createdAt: Date
}
```

## 6. 运营策略

### 6.1 冷启动
1. 内部员工先玩起来，生成第一批讣告内容
2. 邀请 KOL 体验，制造话题
3. 知乎/小红书/微博投放软文

### 6.2 增长黑客
1. 讣告卡片强制带小程序码
2. 复活后邀请好友得额外"复活币"
3. 排行榜每周发奖，刺激竞争

### 6.3 风险控制
1. 设置"轻松模式"，降低死亡焦虑
2. 用户可一键暂停所有通知
3. 明确声明"仅供娱乐"，避免法律风险
